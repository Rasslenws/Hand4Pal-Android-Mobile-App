## Key Architectural Decisions

### 1. **MVVM Pattern**
- **Why**: Separation of concerns, testability, lifecycle awareness
- **How**: ViewModels hold UI state, Fragments observe and react

### 2. **Repository Pattern**
- **Why**: Abstract data sources, single source of truth
- **How**: Repository decides local vs remote data

### 3. **StateFlow over LiveData**
- **Why**: Kotlin-first, better coroutine integration
- **How**: ViewModels expose StateFlow, Fragments collect

### 4. **Retrofit + OkHttp**
- **Why**: Industry standard, powerful, flexible
- **How**: Interceptors for logging and auth

### 5. **DataStore over SharedPreferences**
- **Why**: Type-safe, async, encrypted
- **How**: Store JWT token securely

### 6. **Single Activity + Fragments**
- **Why**: Modern Android navigation, better memory
- **How**: MainActivity + Bottom Nav + Fragments

This architecture ensures the app is scalable, maintainable, testable, and follows Android best practices! ğŸ†
